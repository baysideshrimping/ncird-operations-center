# Deployment Instructions

## Push to GitHub

After creating the repo at https://github.com/new, run:

```bash
cd C:\Users\kevin\ncird-operations-center

# Add GitHub remote
git remote add origin https://github.com/baysideshrimping/ncird-operations-center.git

# Push to GitHub
git branch -M main
git push -u origin main
```

## Deploy to Render

### Option 1: Connect via Render Dashboard (Easiest)

1. **Go to:** https://dashboard.render.com/
2. Click **"New +"** â†’ **"Web Service"**

### Render will auto-detect settings from `render.yaml`:
- **Name:** ncird-operations-center
- **Environment:** Python 3.11.0

```bash

```

```

```

First deploy takes ~5 minutes. Subsequent deploys are faster.

## Updating the App

Whenever you make changes:

```bash
git add .
git commit -m "Description of changes"
git push origin main
```

Render will automatically redeploy on push!

## Troubleshooting

### If build fails:
1. Check build logs in Render dashboard
2. Verify `requirements.txt` is correct
3. Check Python version matches (3.11.0)

### If app won't start:
1. Check start logs in Render dashboard
2. Verify `gunicorn app:app` command
3. Check for port binding issues

### Data persistence:
- Render free tier restarts periodically
- Data in `data/submissions.json` will be lost on restart
- For production, use a database (PostgreSQL, MongoDB)

## Free Tier Limitations

Render free tier:
- Spins down after 15 min of inactivity
- Takes ~30 seconds to wake up
- Restarts clear file-based data
- Good for demos, not production

## Upgrade for Production

For production use:
1. Upgrade to paid tier ($7/month)
2. Add PostgreSQL database
3. Replace JSON storage with DB
4. Add persistent storage
